#Advice from review
1. Add **?** to the end of the method name if the method return a boolean value
2. Try not to write **unecessary comment** can make you write the code more expressive
3. Try to make your code more **logic and clear**
4. Try to make the method do **one thing**
5. Try not **DRY**
6. Leave only **one blank line** between two method or class or logic block
7. Remember to delete the **unecessary blank lines**
8. It is important to signify if the variable is singular or plural by using the varaible name correctly
ex: suit ro suits
9. **Do not define lomg method!!!!**
